# Stage 1 v5 vs v6 对比

## 关键差异
- **v5**: 无 `avg_ep_len` 输出
- **v6**: 加了 `avg_ep_len` 输出,每 step 报告

## 训练曲线

| Step | v5 Return | v6 Return | v6 ep_len | 状态 |
|------|-----------|-----------|-----------|------|
| 100  | 13.35     | 12.05     | 51.5      | 初始(常摔倒) |
| 200  | 16.74     | 19.46     | 84.3      | 学习中 |
| 300  | 24.47     | 32.16     | 138.3     | 改善 |
| 500  | 31.80     | 43.67     | 190.0     | 学会平衡 |
| 600  | 36.60     | **54.49** | **236.9** | **峰值**(稳) |
| 700  | 31.42     | 35.93     | 155.8     | ⚠️ 下降 |
| 900  | 64.92     | 38.33     | 165.3     | ⚠️ 退化 |

## 分析

✅ **v6 加 ep_len 监控很关键** — 能看出训练稳定性
- ep_len 100→600: 51→237(从摔倒到稳定)
- 训练最高峰: step 600(ep_len=237, return=54)

⚠️ **v6 在 step 700-900 退化**:
- ep_len 从 237 降到 165
- return 从 54 降到 38
- 可能原因: Actor 探索激进了 / learning rate 太高 / 数据量不足

## 对比 v5 vs v6(同样 1000 步)

- **v5 @ 900: 64.92**(但 ep_len 未知,可能也短)
- **v6 @ 900: 38.33** + ep_len=165(更稳定但短)

**v6 监控更全面,但训练轨迹更稳定(看 ep_len 趋势)**
**v5 最终 return 更高但可能"虚高"(短 episode 的高 reward)**

## 结论

✅ **加 ep_len 监控是正确方向**
✅ **Step 600 是 v6 最佳 checkpoint**(ep_len=237, return=54)
⚠️ **1000 步太少,需要 50K-100K 步看到稳定学习**

## 建议

1. 跑 50K 步,看完整学习曲线
2. 用最佳 ep_len 的 checkpoint 而非 final 的
3. 添加 early stopping based on ep_len
